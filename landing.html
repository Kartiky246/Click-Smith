Built different

Everything you'd want, nothing you don't

Precision context. Sandboxed execution. Pluggable agents. All on your machine.

🎯
Stable locators, not fragile selectors

ClickSmith ranks locators in priority order. When your Vite build injects data-loc, the agent gets an exact file and line. Otherwise it falls back gracefully — never loses the element.

1source (data-loc)matched
2attr (data-testid, aria-label)fallback
3behavioral (role + text)fallback
4dom (nth-child path)last resort
🛡️
Safe by default

Every run lives in a throwaway git worktree. Your main branch is untouched. Dirty-tree protection refuses runs when you have uncommitted changes — unless you explicitly opt into inplace mode.

plan + worktree
🔌
Bring your own agent

Adapters are config-driven via agents.config.json. Swap agents with a one-line JSON edit. No code changes required.

MCP-native context

The daemon exposes a read-only MCP stdio server. Any MCP-capable agent can call get_session and get_element_by_id directly — no prompt stuffing.

MCP stdio
👁️
Point, don't describe

Multi-element selection. Marks tray shows #1, #2…. The agent sees tag, text, role, nearby labels — not a vague English description.

🏠
100% local-first

No cloud services. No telemetry. Every bundle, screenshot, log, MCP call, and agent command stays on your machine. The daemon only listens on 127.0.0.1.

zero outbound
Get started

Up and running in 4 steps

Node ≥ 20, git ≥ 2.20, Chrome or Firefox. That's all.

1
Run the installer

Auto-detects Vite, Webpack, Rollup, esbuild, or Rspack. Wires up the unplugin for stable data-loc injection.

$ pnpm dlx create-clicksmith
2
Start the daemon

Fastify HTTP + WebSocket + MCP stdio. Runs at localhost:8722. Keep it running in a terminal tab.

$ pnpm clicksmith daemon
3
Install the browser extension

Available for Chrome and Firefox. Talks only to 127.0.0.1 — never phones home.

Chrome Web Store Firefox Add-ons
4
Alt+Click anything

Open your dev server in the browser. Hold Alt and click any element. The overlay appears. Describe your change. Hit Submit.

What happens under the hood
Every submit goes through the same safe pipeline. Your main branch is the last thing touched.
📸
Element captured
Tag, text, role, stable locator, and a screenshot thumbnail bundled into a session.
🌿
Worktree created
Daemon spawns a throwaway git worktree at .clicksmith/worktrees/<runId>. Your main tree is frozen.
main tree untouched
🤖
Agent runs in sandbox
Claude Code (or your agent) reads the bundle via MCP, edits the sandboxed copy, streams logs back to the overlay.
🔍
You review the diff
Inspect the plan and patch in your editor. The worktree sits there untouched until you decide.
✅
Apply (or discard)
Click Apply → 3-way merge back to main, commit created, worktree cleaned up. Or discard with no trace.
revert metadata recorded
