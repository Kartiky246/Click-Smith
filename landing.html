Why it matters

The biggest bottleneck in AI-assisted dev isn't writing — it's locating.

Before an agent can change anything, it has to find the right file, the right component, the right line. That costs tokens. That costs time. That's where most failures happen.

ClickSmith eliminates that search entirely. Instead of vague descriptions and repeated trial-and-error cycles, you hand the agent an exact source location — file, line, element tree, surrounding context — straight from the running UI. The agent starts at the answer, not the question.

Combined with plan-first, isolated worktrees and support for Claude Code, Cursor, and Codex, the result is changes that are faster, cheaper, and right the first time.

Stable locators, not fragile selectors

ClickSmith ranks locators in priority order. When your Vite build injects data-loc, the agent gets an exact file and line. Otherwise it falls back gracefully — never loses the element.

Get started

Up and running in 4 steps

Node ≥ 20, git ≥ 2.20, Chrome or Firefox. That's all.

1
Run the installer

Auto-detects Vite, Webpack, Rollup, esbuild, or Rspack. Wires up the unplugin for stable data-loc injection.

$ pnpm dlx create-clicksmith
2
Install dependencies

Install the packages the wizard added to your project. Choose the package manager you're already using.

$ pnpm install
3
Start the daemon

Fastify HTTP + WebSocket + MCP stdio. Runs at localhost:8722. Keep it running in a terminal tab.

$ pnpm clicksmith daemon
4
Install the browser extension

Available for Chrome and Firefox. Talks only to 127.0.0.1 — never phones home.

Chrome Web Store Firefox Add-ons
5
Alt+Click anything

Open your dev server. Hold Alt and click any element. The overlay appears. Describe your change. Hit Submit. Done.
